### 1. A Dor Real: Por que o Mercado Paga por Isso?

O mercado de **Real World Assets (RWA)** deve atingir **$16 trilhões** até 2030 (Banco de Investimento Goldman Sachs). Mas o gargalo é enorme:

*   **O Problema:** Um banco quer tokenizar um prédio de apartamentos em Lisboa. Eles emitem tokens no Ethereum.
*   **O Risco:** Se o token for transferido para um endereço de um "hackeador" ou de uma pessoa em lista de sanções (AML), o banco perde a licença ou enfrenta processos criminais.
*   **A Falha Atual:** Tokens padrão (ERC-20) são como dinheiro em espécie: se você entregar para alguém, não pode pegar de volta. As empresas não podem arriscar isso.
*   **A Solução:** Um token que **só pode ser enviado para quem tem identidade verificada**.

**Sua Solução Responde a:**
1.  **Conformidade Regulatória (MiCA na Europa):** A Europa está implementando regras rígidas (MiCA - Markets in Crypto-Assets). Sua solução é "compliant" por design.
2.  **Segurança Jurídica:** Garante que o ativo nunca caia na mão errada.
3.  **Interoperabilidade Corporativa:** Permite que bancos (que usam Java/Legado) conversem com a Blockchain moderna.

### O Problema do "Deploy Gratuito" vs. "Blockchain Pública"

Você quer hospedar o Java no **AWS Elastic Beanstalk** e o Frontend na **Vercel**. Isso é ótimo e gratuito (dentro dos limites free tier).
**MAS:** Onde o contrato inteligente (Solidity) vai rodar?
1.  **Local (Anvil/Hardhat):** Não funciona para o público (o recrutador não tem sua máquina rodando o nó local).
2.  **Mainnet (Ethereum Real):** Custa dinheiro real (ETH) para deploy e transação.
3.  **Testnet (Sepolia/Goerli):** **É a solução.** É gratuita, pública e funciona exatamente como a real.

**Estratégia Definitiva:**
*   **Smart Contract:** Deploy na **Testnet Sepolia** (Grátis, usa ETH de "faucet").
*   **Backend (Java):** Elastic Beanstalk (Free Tier) ou **Render/Railway** (Mais fácil para DBs e conexões persistentes, free tier generoso).
*   **Frontend (Next.js):** Vercel (Grátis e perfeito para Next.js).
*   **Oráculo/Validação:** Simulado no seu backend (não precisa de Chainlink agora para o portfólio, o backend Java faz a "validação" e assina a transação).

### Passo a Passo: Como Publicar 100% Grátis e Funcional

#### **1. Deploy do Contrato (Sepolia Testnet)**
Você não precisa pagar nada, apenas pedir "ETH de teste".

1.  **Peça ETH Sepolia:**
    *   Vá em: [Sepolia Faucet](https://sepoliafaucet.com/) ou [Alchemy Sepolia Faucet](https://sepoliafaucet.com/).
    *   Cole sua carteira MetaMask e pegue ETH grátis.
2.  **Configure o Deploy:**
    *   No seu script de deploy do Foundry (`script/DeployToken.s.sol`), adicione a URL da RPC da Sepolia (ex: `https://eth-sepolia.g.alchemy.com/v2/API_KEY` - crie uma chave grátis na Alchemy ou Infura).
    *   Rode:
        ```bash
        forge script script/DeployToken.s.sol --rpc-url https://eth-sepolia.g.alchemy.com/v2/SEU_KEY --broadcast --verify
        ```
    *   *O `--verify` envia o código para o Etherscan, fazendo o contrato aparecer com o nome "PermissionedToken" e o source code visível.*
3.  **Resultado:** Você terá um endereço do contrato (ex: `0x123...abc`) que qualquer pessoa pode ver no [Etherscan Sepolia](https://sepolia.etherscan.io/).

#### **2. Backend (Java Spring Boot)**
O **Elastic Beanstalk** é ótimo, mas configurar variáveis de ambiente e conexões com a blockchain pode ser trabalhoso para iniciantes. Para um portfólio, recomendo o **Render** ou **Railway** (são mais simples, suportam Java/Gradle nativamente e têm free tier).

*Opção A: Render (Recomendado)*
1.  Crie conta no [Render.com](https://render.com).
2.  Conecte seu repositório GitHub.
3.  Crie um **Web Service**.
4.  **Command de Build:** `./mvnw clean package` (ou `mvn package`).
5.  **Command de Start:** `java -jar target/rwa-backend.jar`.
6.  **Variáveis de Ambiente (Crucial):**
    *   `WEB3J_URL`: `https://eth-sepolia.g.alchemy.com/v2/SEU_KEY`
    *   `WEB3J_PRIVATE_KEY`: (Sua chave privada do Admin - **NUNCA** commit no GitHub, use o painel do Render).
    *   `APP_CONTRACT_ADDRESS`: (O endereço do contrato que você deployou).
7.  **Deploy:** O Render vai subir seu backend em `https://seu-projeto-java.onrender.com`.

*Nota: O Elastic Beanstalk também funciona, mas exige configurar VPC e Security Groups para permitir saída para a internet (RPC). O Render é "zero config" para isso.*

#### **3. Frontend (Next.js)**
1.  No [Vercel](https://vercel.com), conecte seu repositório GitHub.
2.  Configure as variáveis de ambiente:
    *   `NEXT_PUBLIC_API_URL`: `https://seu-projeto-java.onrender.com/api/kyc/approve`
3.  Deploy automático. O Vercel vai gerar um link como `https://rwa-portfolio.vercel.app`.

#### **4. O "Pulo do Gato" da Funcionalidade**
Para que o recrutador veja tudo funcionando:
1.  No Frontend, configure a **Wagmi** para usar a rede **Sepolia** (não Hardhat local).
2.  Peça ao recrutador para instalar a MetaMask e adicionar a rede Sepolia (o Vercel pode tentar detectar e pedir para trocar de rede).
3.  **O Fluxo:**
    *   Recrutador entra no link do Vercel.
    *   Conecta a MetaMask (com ETH Sepolia).
    *   Faz upload de um PDF.
    *   Clica em "Aprovar".
    *   O Backend Java (Render) valida, assina e envia para Sepolia.
    *   O Frontend mostra "Sucesso" com link para o Etherscan.
    *   Recrutador clica no link e vê o evento `IdentityAdded` na blockchain.
